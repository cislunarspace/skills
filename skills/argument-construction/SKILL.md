# Argument Construction

帮助用户构建严谨的研究论证，包括提出观点、提供理由、证据、承认与回应、论据。

## 步骤

### 1. 提出观点

观点是研究报告的核心，也是研究问题的答案：

- **观点类型**：概念性观点（要求读者相信什么）或实际性观点（要求读者做什么）
- **明确具体**：使用具体明确的语言和逻辑
- **重要意义**：读者接受观点后需要改变多少其他信念

### 2. 提供理由

理由说明读者为什么应接受该观点：

- **连接观点**：用"因为"连接观点与理由
- **组织论证**：理由勾勒出论证的逻辑和报告的轮廓
- **支持理由**：理由可以被更多的理由支持

### 3. 提供证据

证据是读者认为是事实的说法：

- **正确报告**：确保证据准确、充分、具有代表性
- **恰如其分**：根据领域标准，提供适当精确度的证据
- **显示关联**：加上解说的理由来介绍复杂的证据

### 4. 承认与回应

承认并回应读者可能提出的问题：

- **质疑自己的论证**：像读者一样检验自己的论证
- **找出替代方案**：从原始资料中找出不同的观点
- **决定承认什么**：选择最重要的替代方案来回应

### 5. 提供论据

论据是连接理由和观点的一般性原理：

- **何时陈述**：当读者可能质疑理由和观点之间的关联性时
- **检验论据**：论据是否正确、是否适用于理由和观点
- **挑战论据**：针对不同类型的论据使用不同的挑战策略

## 边界情况

| 情况 | 处理方式 |
|------|----------|
| 论证不严谨 | 使用分镜表组织论证，检验逻辑连接 |
| 证据不足 | 寻找更多证据，承认证据的限制 |
| 忽略反对意见 | 预想读者的问题，主动承认和回应 |
| 论据有争议 | 提出支持论据的论证，承认替代论据 |
| 观点太武断 | 添加限定条件，使用模糊限制语 |

## Checkpoint

- 在用户提出观点后，确认观点是否明确具体
- 在用户提供理由后，确认理由是否支持观点
- 在用户提供证据后，确认证据是否可靠充分
- 在用户回应反对意见后，确认回应是否有效

## 完成条件

- 用户了解论证的五个要素
- 用户能够构建严谨的研究论证
- 用户知道如何承认和回应反对意见
- 用户能够检验和辩护自己的论据
