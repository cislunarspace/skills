# 术语表（GLOSSARY.md）格式

`GLOSSARY.md` 是这个教学工作区的规范用语。所有的讲解、练习和学习记录都要遵守它的术语。建术语表本身就是学习的一部分：把一个概念压成一句紧凑的定义，是用户理解了它的证据。

## 结构

```md
# {主题} 术语表

{一两句话，描述本表覆盖的主题。}

## 术语

**Hypertrophy（肌肉肥大）**:
肌肉在反复训练中，由机械张力和代谢压力驱动的生长。
_避免_：Bulking、getting big

**Progressive overload（渐进超负荷）**:
随着时间推移，系统地通过负荷、训练量或强度增加对肌肉的要求。
_避免_：Pushing harder、levelling up

**RPE（Rate of Perceived Exertion，自感用力程度）**:
给一组动作的吃力程度打 1–10 分的自评，10 是力竭，8 是大约还能再做两个。
_避免_：Effort score、intensity rating
```

## 规矩

- **只在用户理解了一个术语时才把它加进来。** 术语表是压缩后知识的记录，不是用户读来入门的字典。用户刚接触一个概念，要等他能正确使用它，再把它提到这里。
- **要有主张。** 同一概念若有多个叫法，挑最好的那个，其余列为应避免的别名。语言就是这样压缩的。
- **定义要紧凑。** 一两句。定义这个术语**是什么**，不说它做什么、也不说怎么做。
- **定义内部也要用术语表里已有的术语。** 一旦某个术语进了表，到处优先用它，包括在其它定义内部。这正是日后理解复杂术语的关键。
- **必要时分组到小标题下。** 当自然的簇浮现出来时（比如 `## 解剖`、`## 训练安排`）。术语之间本就相通时，平铺也可以。
- **显式标注歧义。** 如果一个术语在更大范围内被用得很松，写明这里的取法："本工作区里，'set'一律指工作组——热身组另算。"
- **理解加深了就修订。** 用户第一周写下的定义，到第六周可能是错的。原地更新，不要留下过时的条目。
