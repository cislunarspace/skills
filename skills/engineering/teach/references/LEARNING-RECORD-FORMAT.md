# 学习记录格式

学习记录放在 `./learning-records/` 下，顺序编号：`0001-slug.md`、`0002-slug.md`，依次递增。目录按需创建——只有写第一条记录时才建。

它们是教学里 ADR 的对应物：记录非显然的教训、关键洞见、用户声明的已有知识，这些东西会左右未来的会话。它们用来推算最近发展区。

## 模板

```md
# {学到了什么、或确立了什么的简短标题}

{1-3 句：学到了什么（或确立了什么已有知识），以及为什么它对未来的会话重要。}
```

这就是全部格式。一条学习记录可以只是一个自然段。它的价值在于记下"现在已知这件事"以及"它如何改变了下一步该教什么"——而不在于把章节填满。

## 可选章节

只有真正增加价值时才加。大多数记录用不上。

- **Status** frontmatter（`active | superseded by LR-NNNN`）——当某个早先的理解被发现是错的、被替换掉时有用。
- **Evidence** ——用户是如何展现这个理解的（答对了某个问题、完成了某个练习、引用了过往经验）。当这条声明日后可能被重新审视时有用。
- **Implications** ——这件事为未来会话解锁了什么、或排除了什么。在它非显然时值得记。

## 编号

扫一遍 `./learning-records/`，找到当前最大的编号，加一。

## 什么时候写学习记录

下列任一情况成立时写一条：

1. **用户展现了对某件非显然之事的真正理解**——不只是接触过，而是有证据表明他能正确运用。这为下一步该教什么设下了新的下限。
2. **用户透露了已有知识**——"我已经懂 X。"记下来，未来会话就不会重教。同时记下他声称的**深度**。
3. **一个误解被纠正**——用户此前相信了某件错的事，现在看清了。这类记录价值很高：它们能预测相关主题未来的绊脚石。
4. **任务因学习而转移**——用户发现自己真正在意的，和原来以为的不一样。交叉链接到 `MISSION.md` 并更新它。

### 什么不够格

- 仅仅是讲过的内容。讲过不等于学会。等有证据再说。
- 已经在术语表（见 GLOSSARY-FORMAT.md）里作为词条定义简洁记下的东西。不要重复。
- 逐次会话的活动流水。学习记录不是日记——它是决策级的洞见。

## 废止

当一条后来的记录和早先的记录冲突（用户的理解加深或纠正了），把旧记录标上 `Status: superseded by LR-NNNN`，不要删掉它。理解如何演进，本身也是有用的信号。
