# RESOURCES.md 格式

`RESOURCES.md` 是这个主题经过精选的可信资源集合。讲解的知识应当从这里来，而不是凭参数化记忆瞎猜。智慧来自这里列出的社群。

## 结构

```md
# {主题} 资源

## 知识

- [书籍：《力量训练的科学与实践》— Zatsiorsky & Kraemer](https://example.com)
  训练安排与适应的基础教材。用途：涉及周期化、恢复、强度区间的内容。
- [文章："How Much Should I Train?" — Greg Nuckols (Stronger By Science)](https://example.com)
  对训练量里程碑的循证综述。用途：每个肌群每周的目标组数。

## 智慧（社群）

- [r/weightroom](https://reddit.com/r/weightroom)
  高信噪比的 subreddit， moderated 抵制 bro-science。用途：训练计划点评、平台期排障。
- 本地：{某健身房} 周二力量课
  用途：对动作的实时教练反馈。
```

## 规矩

- **只收高可信度的。** 优先选一手资料、公认的专家、同行评审的工作、有强版规的社群。如果一份资源其实是披着教育外衣的营销，不要收。
- **每条都要加注。** 一条裸链接三个月后就废了。加一行：它覆盖什么，什么时候该去找它。
- **按知识 / 智慧分组。** 与 [SKILL.md](../SKILL.md) 的理念对应。一份资源只出现在一组里也完全可以。
- **把缺口显式写出来。** 如果任务需要的某个领域找不到好资源，写一节 `## 缺口` 列出缺了什么。这驱动后续的搜索。
- **毫不留情地修剪。** 一份资源被发现是错的、浅的、或偏离任务的，应当删掉，而不是埋在那里。五个锋利的资源胜过三十个平庸的。
- **记录对社群的偏好。** 如果用户已明确不想加入社群，在这里记下，未来会话就不会反复提议。
