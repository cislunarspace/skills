# Handoff

总结当前对话，生成交接文档存到系统临时目录，文件名 `handoff-<简要主题>.md`。

## 文档结构

生成的文档必须包含以下节：

- **下一步建议**：待办任务列表，每个任务写明要改的文件、做什么、验收标准。列出接手 agent 应当调用的技能。
- **依赖关系**：任务之间的先后依赖，标明哪些可以并行。
- **关联 issue**：涉及的 issue 编号（`#N`）或 URL。

## 引用已有工件

PRD、计划、ADR 等文档，以及 commit 历史、当前 diff 里已记录的内容，用路径或 URL 引用，不要复制。

## 脱敏要求

API key、密码、个人身份信息一律抹掉。

## Checkpoint

- 用户传入的参数视为对下次会话重点的说明，据此裁剪文档内容。
- 参数不清晰时，根据当前对话上下文推断下一步重点。
- 文档生成后直接输出文件路径，无需用户确认。

## 完成条件

- 文档已生成到系统临时目录
- 文档包含所有必需节（下一步建议、依赖关系、关联 issue）
- 敏感信息已脱敏
- 文件名符合 `handoff-<主题>.md` 格式
