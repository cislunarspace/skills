# Visual Presentation

帮助用户撰写导言与结论，以视觉方式传达证据，修改文体风格。

## 步骤

### 1. 撰写导言

导言应该让读者相信报告值得阅读：

- **建立语境**：提供必要的背景信息
- **陈述难题**：说明研究问题的重要性
- **概述解决方案**：简要介绍主要观点
- **吸引读者**：回答"那又怎样"的问题

### 2. 撰写结论

结论应该让读者理解论证的意义：

- **重述主要观点**：以不同的方式重申核心论点
- **总结支持**：概述主要理由和证据
- **讨论意义**：说明研究的影响和应用
- **提出建议**：为未来研究或行动提供方向

### 3. 以视觉方式传达证据

使用视觉元素清晰地呈现复杂数据：

- **表格**：呈现量化的数据
- **曲线图**：显示趋势和变化
- **条形图**：比较不同类别
- **图片**：展示物体和事件

### 4. 修改文体风格

让报告清楚、直接而且易读：

- **简洁明了**：删除不必要的词语
- **主动语态**：使用主动语态而非被动语态
- **具体名词**：使用具体名词而非抽象概念
- **平行结构**：使用平行的语法结构

## 边界情况

| 情况 | 处理方式 |
|------|----------|
| 导言不吸引人 | 从读者关心的问题开始，说明研究的重要性 |
| 结论不有力 | 重述主要观点，讨论研究的意义 |
| 视觉元素不清晰 | 添加标签、标题、说明，确保易于理解 |
| 文体风格不佳 | 删除冗余，使用主动语态，简化句子 |
| 证据太复杂 | 使用视觉元素，添加解释说明 |

## Checkpoint

- 在用户撰写导言后，确认导言是否吸引读者
- 在用户撰写结论后，确认结论是否有力
- 在用户使用视觉元素后，确认元素是否清晰易懂
- 在用户修改文体风格后，确认风格是否清楚直接

## 完成条件

- 用户能够撰写吸引人的导言
- 用户能够撰写有力的结论
- 用户知道如何使用视觉元素传达证据
- 用户能够修改文体风格，使报告清楚易读
