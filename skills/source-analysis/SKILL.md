# Source Analysis

帮用户高效阅读资料，做好注释，区分自己的想法和原始资料。

## 步骤

### 1. 选择阅读策略

根据目的选择阅读方式：
- **找难题**：快速浏览，寻找困惑、矛盾、遗漏之处
- **学论证**：仔细阅读，记录论证结构和证据类型
- **找证据**：精确阅读，完整记录数据和出处

让用户先确认阅读目的，再选择策略。

### 2. 做有效注释

教用户记录四类信息：
- **直接引用**：用引号标注，记录页码
- **改写概述**：用自己的话重述，注明出处
- **自己的想法**：与原始资料明确区分
- **元信息**：作者、标题、出版信息、索书码

强调：自己的想法必须和原始资料分开记录。

### 3. 记录论证结构

如果用户在学习论证，帮他们记录：
- 观点是什么？
- 用什么理由支持？
- 用什么证据支持？
- 作者如何回应反对意见？

用问题引导，而不是直接告诉用户答案。

### 4. 验证注释完整性

让用户检查：
- 是否记录了所有必要信息（能重新找到这份资料）？
- 是否区分了直接引用、改写和自己的想法？
- 是否记录了页码和出处？

如果有遗漏，让用户补充。

## 边界情况

| 情况 | 处理方式 |
|------|----------|
| 资料太难理解 | 建议先读概述性资料，再读原始文献 |
| 不确定是否抄袭 | 建议：用自己的话重述，不看原文 |
| 注释太乱 | 建议用统一格式，分类记录 |
| 资料有明显偏见 | 建议记录偏见，寻找对立观点 |

## Checkpoint

- 确认用户选对了阅读策略
- 确认用户能区分四类信息
- 确认注释包含重新找到资料所需的全部信息
