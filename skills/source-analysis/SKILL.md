# Source Analysis

帮助用户批判性地阅读和分析原始资料，包括阅读策略、做注释和记录发现。

## 步骤

### 1. 阅读以找出难题

当你在拟出难题或问题时碰到困难，借着阅读来找出难题或问题：

- 寻找那些让你感到困惑、似乎不正确或过于简化的观点
- 寻找其他人忽略或没有继续追查的数据
- 快速浏览论文的结论部分，作者常在该处指出尚未解决的议题

### 2. 阅读以提出论证

使用原始资料的方式是借用其论证的逻辑：

- 找个类似的论证来当范本
- 记录论证的大纲，并记下用以支持它的证据种类
- 在注释中把特定原始资料提出的论点拟成一个问题

### 3. 阅读以寻找证据

最常见的查阅原始资料的理由是找到可作为证据的资料：

- 尽可能完整而正确地记录证据，并详述该原始资料
- 如果间接引用数据，应该设法找出初级原始资料
- 确保你明白如何公正地报告证据

### 4. 保留你的发现

当找到颇具参考价值的原始资料时，你得有目的地仔细加以阅读：

- **记录完整的参考文献数据**：作者、标题、编辑者、版本、卷期等
- **写下完整的注释**：清楚地区分直接引用、改写、摘录和自己的想法
- **厘清著作权的归属**：将自己的话语与原始资料区分清楚
- **厘清前后文的语境**：记录论证的重要陈述，与条件限制区分开来

## 边界情况

| 情况 | 处理方式 |
|------|----------|
| 资料难以理解 | 放慢速度，重复阅读，寻求帮助 |
| 资料有偏见 | 广泛阅读，寻找不同观点，保持批判性 |
| 资料过时 | 寻找最新研究，检查引用的论文日期 |
| 不确定是否抄袭 | 咨询老师，尽量注明出处 |
| 注释不完整 | 使用标准格式，记录所有必要信息 |

## Checkpoint

- 在用户阅读资料后，确认他们是否理解资料内容
- 在用户做注释后，确认注释是否完整准确
- 在用户使用资料前，确认他们是否区分了自己的想法和原始资料

## 完成条件

- 用户了解三种阅读目的
- 用户能够批判性地阅读和分析资料
- 用户知道如何做完整的注释
- 用户能够区分自己的想法和原始资料
