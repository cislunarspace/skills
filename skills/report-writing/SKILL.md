# Report Writing

帮助用户规划和撰写研究报告，包括规划草稿、撰写初稿、修改组织架构和论证。

## 步骤

### 1. 规划草稿

在撰写草稿前，先思考一番，描绘出一个尝试性的计划：

- **决定主要论点**：在导言结尾处或结论开头处陈述
- **规划暂时性导言**：提供语境、研究问题、问题后果、假设性解答
- **组织报告主体**：根据论证逻辑安排理由和证据

### 2. 撰写初稿

选择适合你的写作方式：

- **快而杂乱**：文思泉涌、下笔神速，不担心风格问题
- **慢而清晰**：字斟句酌地下笔，每个字句都确定下来

### 3. 修改组织架构

像读者那样思考，从整体到部分进行修改：

- **分析外在架构**：确认导言和结论的位置
- **确认主要章节**：区分导言与论点句
- **检查主题概念**：找出关键主题在整篇文章里的位置

### 4. 修改论证

检验论证的每个部分：

- **检验观点**：观点是否明确具体
- **检验理由**：理由是否支持观点
- **检验证据**：证据是否可靠充分
- **检验回应**：是否回应了反对意见

## 边界情况

| 情况 | 处理方式 |
|------|----------|
| 不知道如何开始 | 使用暂时性导言，从最容易的部分开始 |
| 组织混乱 | 使用分镜表，尝试不同的排列顺序 |
| 论证不清晰 | 检验逻辑连接，添加论据 |
| 抄袭风险 | 区分自己的想法和原始资料，正确引用 |
| 写作瓶颈 | 做个标记，核对大纲，继续写下去 |

## Checkpoint

- 在用户规划草稿后，确认规划是否合理
- 在用户撰写初稿后，确认是否包含所有必要部分
- 在用户修改组织架构后，确认架构是否清晰
- 在用户修改论证后，确认论证是否严谨

## 完成条件

- 用户了解研究报告的组织结构
- 用户能够规划和撰写初稿
- 用户知道如何修改组织架构和论证
- 用户能够避免抄袭，正确引用资料
