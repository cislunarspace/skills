# Problem Identification

帮助用户从兴趣中找到题目，从题目中提出问题，从问题中界定难题。

## 步骤

### 1. 从兴趣到题目

研究题目是个严密界定的兴趣，严密到足以想象自己成为这方面的专家：

- **列出兴趣**：列出两三个你想去探究的兴趣
- **浏览参考书目**：查看一般参考文献的标题，寻找灵感
- **缩小范围**：将广泛的兴趣缩小到具体的题目

### 2. 从广泛的到明确的题目

使用增加特定文字及词组来限定主题：

- **添加动作**：使用冲突、描述、贡献、发展等词
- **形成观点**：让题目接近一个让读者可能觉得重要的观点
- **避免过窄**：不要把题目变得太狭隘，以至于无法找到足够的数据

### 3. 从明确的题目到问题

用许多提问来质疑题目：

- **标准问题**：什么人、什么事情、什么时间、什么地点
- **分析性问题**：结构、历史、类别、价值
- **评价问题**：将小问题汇聚成更有意义的问题

### 4. 从问题到难题

研究难题有两个组成部分：

1. **情境**：对事物不知道或不了解的描述
2. **后果**：读者不知道或不了解的其他事物，但具有重要意义

使用三步骤公式：
1. 题目：我正在研究……
2. 问题：因为我想要找出……
3. 重要意义：为了帮助读者了解……

## 边界情况

| 情况 | 处理方式 |
|------|----------|
| 找不到题目 | 从个人兴趣出发，浏览参考书目，咨询老师 |
| 题目太宽泛 | 添加特定文字，限定范围，聚焦问题 |
| 问题太宽泛 | 将小问题汇聚成更有意义的问题 |
| 难题不明确 | 使用三步骤公式，从问题推导出重要意义 |
| 找不到重要意义 | 思考问题对读者的影响，回答"那又怎样" |

## Checkpoint

- 在用户确定题目后，确认题目是否足够具体
- 在用户提出问题后，确认问题是否值得探究
- 在用户界定难题后，确认难题是否具有重要意义

## 完成条件

- 用户确定了具体的研究题目
- 用户提出了值得探究的研究问题
- 用户界定了具有重要意义的研究难题
- 用户能够使用三步骤公式描述研究
