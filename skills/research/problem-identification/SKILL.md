# Problem Identification

帮用户从模糊兴趣缩小到具体题目，再从题目中提炼出研究问题和难题。

## 步骤

### 1. 从兴趣到题目

让用户列出2-3个感兴趣的方向，然后缩小范围：
- 如果题目能用四五个字概括（如"商用飞机的历史"），太宽泛
- 使用动词性名词（冲突、贡献、发展）让题目更具体
- 目标：得到一个能想象自己成为专家的题目

示例：
- "商用飞机的历史" → "军事对DC-3民航机在早期商用飞行的发展的重要贡献"

### 2. 从题目到问题

用四类问题质疑题目：
- **结构**：由哪些部分组成？如何成为更大系统的一部分？
- **历史**：如何随时间改变？在更大历史中扮演什么角色？
- **类别**：属于哪一类？与其他类似事物有何不同？
- **价值**：反映出什么价值？多好或多糟？

把小问题汇聚成1-2个有意义的问题。

### 3. 从问题到难题

使用三步骤公式：
1. **题目**：我正在研究……
2. **问题**：因为我想要找出……
3. **重要意义**：为了帮助读者了解……

让用户填空，然后追问"那又怎样"，直到能说服读者在乎。

### 4. 验证难题价值

问用户：
- 如果无法回答这个问题，会有什么后果？
- 这个后果对读者重要吗？
- 读者会不会问"那又怎样"？

如果用户答不上来，建议缩小题目或换方向。

## 相关技能

**上游**（需要先完成）：
- `/research-planning`：理清研究方向，建立与读者的联系

**下游**（接下来该用）：
- `/source-collection`：寻找可靠资料，判断资料是否值得信赖

**并行**（贯穿全程）：
- `/research-ethics`：避免抄袭，正确引用，保持学术诚信

## 边界情况

| 情况 | 处理方式 |
|------|----------|
| 用户找不到任何兴趣 | 建议浏览百科全书、期刊标题、新闻 |
| 题目一直太宽泛 | 用具体示例演示如何缩小 |
| 问题太多太散 | 帮用户归类，选出最核心的1-2个 |
| 用户说不清重要意义 | 追问"那又怎样"，直到答不上来为止 |

## Checkpoint

- 确认用户能用一句话说清题目
- 确认用户有明确的研究问题
- 确认用户能完成三步骤公式
