# Visual Presentation

帮用户写出吸引人的导言、有力的结论，用视觉元素呈现数据，改进文体风格。

## 步骤

### 1. 完善导言

检查导言是否回答了四个问题：
- 为什么这个题目重要？（语境）
- 我们不知道什么？（难题）
- 不知道会怎样？（后果）
- 我发现了什么？（答案）

如果导言太泛或太长，建议删减到只保留这四个要素。

### 2. 完善结论

检查结论是否包含：
- 重述主要观点（用不同措辞）
- 总结关键理由和证据
- 讨论研究的意义或应用
- 指出研究的局限或未来方向

避免在结论中引入新证据或新论证。

### 3. 使用视觉元素

问用户需要呈现什么类型的数据：
- **表格**：精确数值、多变量比较
- **曲线图**：趋势变化
- **条形图**：类别比较
- **图片**：展示实物或场景

提醒用户：每个视觉元素都要有标题、标签、说明。

### 4. 改进文体风格

给用户四个原则：
- 删除不必要的词语
- 优先用主动语态
- 用具体名词代替抽象概念
- 平行内容用平行结构

让用户逐段检查，每次只改一个问题。

## 边界情况

| 情况 | 处理方式 |
|------|----------|
| 导言太长 | 建议删减到核心四要素 |
| 结论引入新内容 | 建议移到正文或删除 |
| 视觉元素太复杂 | 建议拆分成多个简单图表 |
| 用户纠结风格 | 建议先完成内容，最后再改风格 |

## Checkpoint

- 确认导言回答了四个核心问题
- 确认结论没有引入新内容
- 确认视觉元素有标题和说明
