# Visual Presentation

根据用户需求，完善研究报告的导言、结论、视觉元素或文体风格。每次聚焦一个方面，识别问题，给出具体改进建议或修改版本。假设用户已有初稿。

## 步骤

根据用户请求选择对应步骤执行。用户可能只需要改进其中一个方面。

### 1. 完善导言

检查用户的导言是否回答了四个问题，标出缺失的要素：
- 为什么这个题目重要？（语境）
- 我们不知道什么？（难题）
- 不知道会怎样？（后果）
- 我发现了什么？（答案）

如果导言太泛或太长，给出删减建议，只保留这四个要素。

### 2. 完善结论

检查用户的结论是否包含以下要素，标出缺失部分并给出补充建议：
- 重述主要观点
- 总结关键理由和证据
- 讨论研究的意义或应用
- 指出研究的局限或未来方向

避免在结论中引入新证据或新论证。

### 3. 使用视觉元素

根据用户内容中的数据类型，建议合适的视觉元素：
- **表格**：精确数值、多变量比较
- **曲线图**：趋势变化
- **条形图**：类别比较
- **图片**：展示实物或场景

提醒用户：每个视觉元素都要有标题、标签、说明。如果用户已有数据但未可视化，直接给出具体方案。

### 4. 改进文体风格

用四个原则检查用户文本，标出所有问题：
- 删除不必要的词语
- 优先用主动语态
- 用具体名词代替抽象概念
- 平行内容用平行结构

如果问题较多，建议分批修改；如果较少，可以直接给出修改版本。

## 边界情况

| 情况 | 处理方式 |
|------|----------|
| 导言太长 | 建议删减到核心四要素 |
| 结论引入新内容 | 建议移到正文或删除 |
| 视觉元素太复杂 | 建议拆分成多个简单图表 |
| 用户对多个风格选项犹豫不决 | 展示 2-3 个具体改写示例，让用户选择偏好方向 |

## Checkpoint

每次改动前展示修改方案，等用户确认后执行。视觉元素的选型建议可以直接给出，不需要等确认。

- 确认导言回答了四个核心问题
- 确认结论没有引入新内容
- 确认视觉元素有标题和说明
