# Report Writing

引导用户完成报告撰写全流程：先规划结构，写暂时性导言，选择写作方式，最后修改架构和论证。每个步骤结束前确认用户理解并完成当前任务。

## 步骤

### 1. 规划报告结构

问用户两个决策问题，解释每个选项的影响：
- **主要论点放在哪**：导言结尾（读者掌控阅读）还是结论开头（读者跟随你）
- **章节顺序**：从旧到新、从简单到复杂、从无争议到有争议

用大纲帮用户排列论证要素。

### 2. 撰写暂时性导言

让用户写四句话：
1. **语境**：关于这个题目，读者已经知道什么
2. **问题**：我们不知道或不了解什么
3. **后果**：不知道这些会有什么损失
4. **答案**：我的研究发现了什么（暂时性的）

用户写完后，检查四句话是否对应语境、问题、后果、答案。如有缺失或不清晰，指出具体问题并要求补充。

这个导言会在修改时重写，但能帮用户开始写作。

### 3. 选择写作方式

问用户偏好哪种：
- **快而杂乱**：先写下来，不担心风格，之后再改
- **慢而清晰**：字斟句酌，每个句子确定后再继续

### 4. 修改组织架构

读用户的草稿，逐项检查以下方面。发现问题时指出具体位置和改进方向：
- 导言是否清楚说明了难题和答案？
- 每个章节是否有明确的论点句？
- 章节顺序是否符合逻辑？
- 承认与回应是否放在合适位置？

### 5. 修改论证

读用户的草稿，检验每个部分。发现论证缺陷时指出具体问题和补救方向：
- 观点是否明确？
- 理由是否支持观点？
- 证据是否可靠充分？
- 是否回应了主要反对意见？

## 相关技能

**上游**（需要先完成）：
- `/argument-construction`：构建完整的论证（观点、理由、证据）

**下游**（接下来该用）：
- `/visual-presentation`：完善导言结论，使用视觉元素，改进文体风格

**并行**（贯穿全程）：
- `/research-ethics`：避免抄袭，正确引用，保持学术诚信

## 边界情况

| 情况 | 处理方式 |
|------|----------|
| 用户不知道如何开始 | 建议先写暂时性导言，从最容易的部分写起 |
| 用户只堆砌资料 | 提醒用户要有自己的观点和论证 |
| 用户纠结风格 | 建议先用"快而杂乱"方式，之后再改 |
| 用户想抄袭 | 明确拒绝协助抄袭行为。引导用户使用 `/research-ethics` 学习正确引用方法 |

## Checkpoint

每完成一个步骤前，确认用户已完成当前任务再继续：
- 步骤 1 结束前确认用户有明确主要论点
- 步骤 2 结束前确认用户写了暂时性导言
- 步骤 3 前确认用户知道初稿不是终稿

## 完成条件

- 用户明确表示不需要进一步修改，或
- 用户已进入 `/visual-presentation` 阶段
