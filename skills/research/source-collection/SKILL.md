# Source Collection

帮用户找到可靠的原始资料，判断资料是否值得信赖。

## 步骤

### 1. 确定资料类型

问用户需要什么类型的资料：
- **初级原始资料**：原始文献、实验数据、访谈记录
- **二级原始资料**：基于初级资料的研究报告、论文
- **三级原始资料**：百科全书、综述文章（只用于入门）

根据研究问题，告诉用户应该优先找哪种。

### 2. 判断资料可靠性

给用户四个检验标准：
- 出版社是否有信誉？（大学出版社通常可靠）
- 是否经过同行审查？
- 作者在领域内是否有声望？
- 资料是否足够新？（不同领域标准不同）

让用户用这四个标准筛选找到的资料。

### 3. 指导搜索路径

按优先级建议：
1. **图书馆**：馆藏目录、专业数据库、研究指南
2. **互联网**：可靠期刊网站、政府数据库、学术机构网站
3. **直接搜集**：访谈专家、问卷调查、实地观察

如果用户不知道从哪开始，建议先咨询图书馆管理员。

### 4. 处理互联网资料

提醒用户互联网资料的风险：
- 优点：更新快、获取方便、有独特数据
- 缺点：没有把关者、质量参差不齐、可能有偏见

给出可用的互联网资料类型：
- 可靠期刊的在线版本
- 政府和学术机构的数据库
- 作为初级原始资料的网络内容

## 相关技能

**上游**（需要先完成）：
- `/problem-identification`：确定研究问题，明确需要什么类型的资料

**下游**（接下来该用）：
- `/source-analysis`：批判性阅读资料，做好注释，区分自己的想法和原始资料

**并行**（贯穿全程）：
- `/research-ethics`：避免抄袭，正确引用，保持学术诚信

## 边界情况

| 情况 | 处理方式 |
|------|----------|
| 用户只用互联网搜索 | 建议先从图书馆开始，互联网作为补充 |
| 找不到相关资料 | 建议换关键词、扩大范围、咨询专家 |
| 资料太多无法筛选 | 帮用户聚焦问题，缩小搜索范围 |
| 资料相互矛盾 | 建议记录不同观点，留待分析时判断 |

## Checkpoint

- 确认用户知道需要哪种类型的资料
- 确认用户能用四个标准判断可靠性
- 确认用户有明确的搜索路径
