# Source Collection

根据用户的研究问题，确定所需资料类型，提供搜索路径，用可靠性标准筛选资料。

## 步骤

### 1. 确定资料类型

确定用户需要的资料类型：
- **初级原始资料**：原始文献、实验数据、访谈记录
- **二级原始资料**：基于初级资料的研究报告、论文
- **三级原始资料**：百科全书、综述文章，用于建立初步认识

根据研究问题推荐优先级。

### 2. 判断资料可靠性

用以下标准评估资料可靠性：
- 出版社信誉
- 同行审查
- 作者声望
- 出版时效

将评估结果告知用户。

### 3. 指导搜索路径

向用户提供搜索路径，按优先级排序：
1. **图书馆资源**：馆藏目录、专业数据库、研究指南
2. **互联网资源**：可靠期刊、政府数据库、学术机构网站
3. **直接搜集**：访谈专家、问卷调查、实地观察

若用户不清楚起点，建议咨询图书馆管理员。

### 4. 处理互联网资料

推荐互联网资料时，优先可靠期刊在线版、政府和学术机构数据库。说明获取方式和可靠性依据。

## 相关技能

**上游**（需要先完成）：
- `/problem-identification`：确定研究问题，明确需要什么类型的资料

**下游**（接下来该用）：
- `/source-analysis`：批判性阅读资料，做好注释，区分自己的想法和原始资料

**并行**（贯穿全程）：
- `/research-ethics`：避免抄袭，正确引用，保持学术诚信

## 边界情况

| 情况 | 处理方式 |
|------|----------|
| 用户只用互联网搜索 | 建议先从图书馆开始，互联网作为补充 |
| 找不到相关资料 | 建议换关键词、扩大范围、咨询专家 |
| 资料太多无法筛选 | 帮用户聚焦问题，缩小搜索范围 |
| 资料相互矛盾 | 建议记录不同观点，留待分析时判断 |

## Checkpoint

每完成一个步骤（确定资料类型、提供搜索路径、给出可靠性评估）后，等用户反馈再继续。

若用户提出的搜索需求超出可公开获取的资源范围（需要付费数据库、专业访谈资源），停下来说明情况。
