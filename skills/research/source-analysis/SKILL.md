# Source Analysis

引导用户选择阅读策略，记录四类信息（直接引用、改写、自己想法、元信息），验证注释完整性。重点：确保用户的想法与原始资料明确区分。

## 步骤

### 1. 选择阅读策略

询问用户阅读目的，根据目的选择阅读方式：
- **识别问题**：快速浏览，寻找困惑、矛盾、遗漏
- **分析论证**：仔细阅读，记录论证结构和证据类型
- **提取证据**：精确阅读，完整记录数据和出处

### 2. 做有效注释

确认用户记录了四类信息：
- **直接引用**：用引号标注，记录页码
- **改写概述**：用自己的话重述，注明出处
- **自己的想法**：与原始资料明确区分（关键）
- **元信息**：作者、标题、出版信息、索书码

### 3. 记录论证结构

如果用户在分析论证，逐一询问：
- 观点是什么？
- 用什么理由支持？
- 用什么证据支持？
- 作者如何回应反对意见？

### 4. 验证注释完整性

检查用户是否完成：
- 记录了所有必要信息（能重新找到这份资料）
- 区分了直接引用、改写和自己的想法
- 记录了页码和出处

有遗漏时，要求用户补充。

## 相关技能

**下游**：
- `/argument-construction`：用分析好的资料构建论证

**并行**：
- `/research-ethics`：避免抄袭，正确引用，保持学术诚信

## 边界情况

| 情况 | 处理方式 |
|------|----------|
| 用户未提供资料路径 | 要求用户提供具体资料信息或路径 |
| 资料格式无法访问 | 询问是否有其他格式版本 |
| 资料太难理解 | 建议先读概述性资料，再读原始文献 |
| 不确定是否抄袭 | 建议：用自己的话重述，不看原文 |
| 注释太乱 | 建议用统一格式，分类记录 |
| 资料有明显偏见 | 建议记录偏见，寻找对立观点 |

## Checkpoint

- 阅读策略选择后，向用户确认目的是否匹配
- 记录完成后，检查是否有标注出处、页码、引号
- 完成前，让用户确认能用元信息重新找到资料

## 完成条件

- 用户确认阅读目的已达成
- 四类信息已记录
- 所有引用含出处和页码
- 自己的想法与原文明确区分
